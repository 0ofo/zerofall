# 烬 ZeroFall

采用 **Native AOT** 发布为单文件可执行程序：**冷启动快**、无需安装 .NET 运行时，双击即用。

面向攻防与情报分析——**类 Burp Suite** 的 HTTP 流量截获、检视与重放，再叠加资产测绘、SQL 分析、AI 助手与交互式终端，统一在一个项目里完成。

打开文件夹即项目；侦察结果、流量记录与终端记录落在项目本地数据库（`.zerofall.db`）中，便于用 SQL 与 AI 在同一工作区内分析。

## 为什么用 烬

- **类 Burp Suite**：内置浏览器 + 代理/拦截，HTTP(S) 流量全量入库；支持检视、重放、Intruder、Decoder/Diff 等 HTTP 工作流，网站树按会话组织请求。
- **Native AOT**：预编译原生二进制，启动迅速，适合频繁开关、多项目切换的日常攻防作业。
- **一张工作台**：测绘、抓包、查库、跑命令、问 AI 不必在多个工具间来回倒数据。

---

## 能做什么

| 能力 | 说明 |
|------|------|
| **HTTP 流量（类 Burp）** | WebView2 + CDP / 代理捕获，历史流量 SQLite 持久化，重放与 Intruder，请求/响应高亮检视 |
| **项目管理** | 打开工作目录，侧边栏文件树，数据文件索引入库 |
| **SQL 分析** | 内置编辑器与结果表格，查询项目数据库 |
| **资产测绘** | FOFA / Hunter / Quake / Shodan 等多源聚合、去重与入库 |
| **AI 助手** | OpenAI 兼容 API，流式对话，内置工具与 MCP，可联动浏览器、终端与项目数据 |
| **交互式终端** | 多会话 PTY，可与 AI 协同 |
| **代理** | 系统/拦截代理，与浏览器及出站请求联动 |

## 环境要求

- **操作系统**：Windows 10/11（x64）
- **WebView2**：[Microsoft Edge WebView2 Runtime](https://developer.microsoft.com/microsoft-edge/webview2/)（浏览器与流量相关功能需要）

发布包为 **Native AOT 单文件**（约百兆级），不依赖系统安装 .NET。

---

## 安装与使用

从本仓库 **Releases** 下载最新 `ZeroFall-*-win-x64.zip`，解压到任意目录，运行其中的 **`ZeroFall.App.exe`**（无需安装 .NET）。

1. **打开文件夹** 选择工作目录  
2. 在 **设置** 中按需配置 AI API、资产测绘 Key、代理等  
3. 新建浏览器标签抓包，或用 SQL / AI / 终端继续分析  

> 使用 AI、资产测绘、MCP 等功能前，请先在设置中完成相关配置。各测绘源 API Key 由用户自行申请，费用与配额以各平台为准。
